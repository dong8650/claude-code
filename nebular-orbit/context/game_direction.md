## 1. 핵심 방향 전환: 타이머 → 이동 횟수 제한

### 타이머를 도입했던 이유 (일반적)

| 이유 | 설명 |
|------|------|
| 긴장감 부여 | 시간 압박 → 스릴 → 몰입 |
| 난이도 조절 | 퍼즐 자체 난이도 낮아도 타이머로 도전감 |
| 무한 힌트 방지 | 천천히 생각하면서 힌트 남용 차단 |
| 점수 차별화 | 빠를수록 고점 → 리플레이 유도 |

### 성운궤도에서 타이머를 넣은 실제 이유 3가지

#### 이유 1. 몰입도 상승
**평가: 절반만 맞습니다**
수학 퍼즐 장르에서 타이머 압박은 몰입이 아니라 **스트레스**로 전환될 수 있습니다. 생각을 요구하는 게임 + 시간 압박은 서로 충돌합니다. 피드백 #1(눈 피로)의 원인 중 하나가 타이머 긴장감일 가능성이 높습니다.

#### 이유 2. 힌트 수익 창출
**평가: 맞는 방향, IAP 기반에서 유효**
타이머 압박 → 힌트 구매 유도 → 수익. 논리는 맞습니다. IAP(인앱결제) 기반에서 유효하며, 실제로 Google Play에 4종 IAP가 구축되어 있어 이 로직은 작동합니다.

#### 이유 3. 세션 종료 트리거 ⭐ (핵심)
**가장 정확한 이유입니다.**

```
타이머 없는 경우:
접속 → 게임 시작 → 천천히 연결 → 영원히 진행 가능
→ 게임오버 없음 → 광고 노출 없음 → 재방문 이유 없음
→ 수익 0, 리텐션 0
```

타이머가 해결하는 것:
- 자연스러운 게임오버 발생
- 게임오버 → 광고 시청 → 재시작 루프
- 재방문 트리거 (목숨/힌트 보충)

### 핵심 질문

> **"성운궤도의 타겟 유저가 스릴을 원하는가, 아니면 힐링을 원하는가?"**

캔디크러쉬, 로열매치 등 대부분의 대형 캐주얼 퍼즐은 **타이머 없이 이동 횟수 제한**을 씁니다. 시간 압박 없이도 충분히 도전감을 줄 수 있기 때문입니다.

### 방식 비교

| 방식 | 세션 종료 | 수익 유도 | 스트레스 | 캐주얼 친화 |
|------|----------|----------|---------|------------|
| **타이머 (현재)** | ✅ | ✅ | ❌ 높음 | ❌ |
| **이동 횟수 제한** | ✅ | ✅ | ✅ 낮음 | ✅ |
| **제한 없음** | ❌ | ❌ | ✅ 없음 | ✅ |

## 2. 이동 횟수 제한 — 정확한 작동 방식

### 핵심 규칙

```
✅ 2+1 = 3 (타겟 일치) → 별 폭발 + 이동 횟수 -1  (9회 남음)
❌ 2+2 = 4 (타겟 불일치) → 아무 일도 없음, 이동 횟수 유지 (10회 그대로)
```

**실패는 무료, 성공할 때만 횟수가 줄어드는** 구조입니다.

### 플레이어 심리 비교

| | 타이머 방식 | 이동 횟수 방식 |
|---|---|---|
| 압박 요소 | "빨리 해야 해" | "신중하게 골라야 해" |
| 잘못된 시도 | 시간 낭비 | 페널티 없음 (자유롭게 탐색) |
| 게임오버 조건 | 시간 소진 | 성공 연결 N회 소진 |
| 전략 | 속도 | 효율 (1회로 더 많이 폭발) |
| 실패 원인 | "시간이 부족했어" (외부 탓) | "내가 비효율적으로 했네" (내 탓) |
| 재도전 의욕 | "짜증나서 그냥 꺼" | "다시 하면 잘 할 수 있을 것 같은데" |
| 구매 의향 | 낮음 | 높음 ("조금만 더 하면 클리어인데") |

### 수익 구조가 더 자연스러운 이유

```
남은 이동 3회... 클리어하려면 5회는 필요한데
→ [+5회 추가 - 힌트팩 사용] 버튼이 자연스럽게 등장
→ hints_10 / hints_50 IAP 상품과 직접 연결
```

타이머의 "광고 보고 +30초"보다 **구매 동기가 훨씬 명확**합니다.

### 추가 설계 포인트 3가지

**1. 남은 횟수 경고**
```
3회 남았을 때 → 숫자 빨간색 + 진동
"마지막 3회입니다! 큰 연결을 노려보세요"
```

**2. 레벨 캘리브레이션**
```
2~3개 연결만 해도 클리어 가능하도록 이동 횟수 여유있게 설정
평균 플레이어 → 2~3회 남기고 클리어
잘하는 플레이어 → 5회 이상 남김
못하는 플레이어 → 0회, Continue 유도
```

**3. Continue 팝업 타이밍**
```
클리어까지 별 3~5개만 더 터뜨리면 될 때만 팝업
"딱 2번만 더 연결하면 클리어!" → 구매 전환율 높음
```

## 3. 개발 구현 범위 (프로덕션 승인 후)

### 타이머 코드 현황
```
총 타이머 관련 코드: ~41곳 참조
핵심 함수: 4개
├── startTimer()       — 타이머 시작
├── updateTimer()      — 매 0.1초 UI 업데이트
├── _onTimerEnd()      — 시간 종료 처리
└── _claimTimeExtend() — "광고 보고 30초 연장"

행성별 timeLimit: 11개 행성 각각 설정
(지구 90초, 달 90초, ... 마지막 행성 70초)
```

### 변경 작업 테이블

| 작업 | 내용 | 난이도 |
|------|------|--------|
| 핵심 함수 4개 교체 | startTimer→startMoves, updateTimer→updateMoves 등 | ⭐⭐ |
| 연결 성공 시점에 moves-- 추가 | 별 폭발 함수 한 곳에 한 줄 추가 | ⭐ |
| `timeLimit` → `moveLimit` 변경 | 11개 행성 숫자만 바꾸면 됨 | ⭐ |
| HTML UI 교체 | 타이머 바 → 이동 횟수 숫자 | ⭐ |
| "광고→+30초" → "광고→+5회" | _onTimerEnd 로직 수정 | ⭐⭐ |
| IAP +5회 연결 | hints_10/hints_50 상품 연동 | ⭐⭐ |
| 밸런싱 | 행성별 적정 이동 횟수 테스트 | ⭐⭐⭐ |

**전체 코드 14,000줄 중 실제 변경은 100~150줄 수준**

### 예상 작업 시간

| 단계 | 시간 |
|------|------|
| 코드 변경 | 3~4시간 |
| 밸런싱 (행성별 횟수 조정) | 2~3시간 |
| 테스트 | 1~2시간 |
| **합계** | **반나절~하루** |

## 6. 게임오버 팝업 설계

이동 횟수 0 소진 시 자동 표시. 3가지 선택지 제공.

```
┌────────────────────────────┐
│     이동 횟수 부족!         │
│  계속하려면 이동 추가하세요  │
├────────┬────────┬──────────┤
│  ⭐×3  │ ₩1,200 │  📺 광고  │
│힌트→+5회│ +15회  │  +3회 무료│
└────────┴────────┴──────────┘
│     처음부터 다시 (무료)     │
```

- 힌트 소비 → 이동 전환: 기존 힌트 재고 소진 유도
- IAP 직결: 게임오버 시점에 결제 의향 최고조
- 광고: 결제 거부 사용자 수익화
- "처음부터" 옵션 항상 제공 → 강압감 제거

## 7. 성운 유니버스 확장 로드맵

### 기존 게임 고도화 vs 교체 평가

| 게임 | 성운 브랜드 적합성 | 장르 경쟁력 | 판단 |
|------|------------------|------------|------|
| 코스모 팜 | ✅ 높음 (우주 농장) | ✅ 아이들 게임 트렌드 | **고도화 → 성운 팜** |
| 젬 블라스트 | ✅ 중간 (별/젬 연결) | ✅ 매치3 최고 인기 장르 | **고도화 → 성운 매치** |
| 리듬 히어로 | ⚠️ 낮음 | ⚠️ 음악 저작권 이슈 | **보류 or → 성운 비트** |
| 초성 퀴즈 킹 | ❌ 낮음 (한국어 특화) | ⚠️ 틈새 장르 | **교체** |

### 제안 라인업

| 슬롯 | 게임명 | 장르 | 방향 |
|------|--------|------|------|
| 1 | **성운궤도** | 수학 퍼즐 | 피드백 반영 고도화 |
| 2 | **성운 팜** | 아이들 클리커 | 코스모팜 리브랜딩 + 오프라인 수익 + 업그레이드 트리 |
| 3 | **성운 매치** | 매치3 퍼즐 | 젬블라스트 → 레벨 50개+ 설계, 목표 다양화 |
| 4 | **성운 머지** | 머지 퍼즐 | 신규 개발 (트렌드 최상위 장르) |
| 5 | **성운 비트** | 리듬 | 리듬히어로 → 자체 우주 BGM 교체 |

### 신규 게임 대박 가능성 TOP 5

#### 🥇 1위 — 성운 수이카 (Cosmo Drop)
**모티브:** 스이카게임 (닌텐도 스위치 수백만 카피, 전 세계 바이럴)

```
운석 떨어뜨리기
→ 같은 크기 닿으면 합체
→ 운석 → 소행성 → 달 → 행성 → 가스행성 → 항성 → 블랙홀 → 은하
```

- 물리 엔진 기반 → 매번 다른 결과 = 무한 재미
- 룰 설명 0초 → 누구나 이해
- 최고점수 경쟁 → 바이럴 공유
- HTML5 + Matter.js로 구현 가능
- **개발 난이도: ⭐⭐ (2주)**

#### 🥈 2위 — 성운 서바이버 (Cosmo Survivors)
**모티브:** Vampire Survivors (Steam 역대급 인디 히트)

```
플레이어는 이동만 / 자동으로 공격 발사
몰려오는 운석/적 처치 → XP 수집 → 업그레이드 선택
30분 생존이 목표
```

- "손 안 떼는" 자동전투 → 중독성 최강
- 업그레이드 조합이 무한 → 높은 리플레이
- 한 판 15~30분 → 유튜브/스트리밍 친화적
- **개발 난이도: ⭐⭐⭐ (4주)**

#### 🥉 3위 — 성운 커넥션 (Cosmo Connections)
**모티브:** NYT Connections (매일 수백만 명 플레이)

```
16개 우주 키워드
→ 4개씩 같은 카테고리로 묶기
→ 예) 행성 / 별자리 / 우주 용어 / 성운궤도 아이템
→ 매일 새 문제 → 매일 재방문
```

- 데일리 퍼즐 = 습관성 재방문
- 결과 공유 (색상 이모지) → 자연 바이럴
- 성운궤도 세계관 연계 최강
- **개발 난이도: ⭐ (3~5일)**

#### 4위 — 성운 디펜스 (Cosmo Defense)
**모티브:** BTD 스타일 타워디펜스
- 행성을 지켜라: 경로로 몰려오는 운석/외계인 처치
- 포탑 설치 + 업그레이드, 웨이브 클리어
- 유튜브 스트리밍 최적 장르
- **개발 난이도: ⭐⭐⭐⭐ (6~8주)**

#### 5위 — 성운 슬링 (Cosmo Sling)
**모티브:** Angry Birds (역대 모바일 최고 히트작)
- 새총으로 행성 발사, 적 행성/구조물 파괴
- 물리 기반 퍼즐, 별 3개 클리어 시스템
- **개발 난이도: ⭐⭐⭐ (3~4주)**
