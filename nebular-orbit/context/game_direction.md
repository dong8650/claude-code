## 1. 핵심 방향 전환: 타이머 → 이동 횟수 제한

### 타이머를 도입했던 이유
1. 타이머를 통한 몰입도 상승 (긴장감)
2. 힌트 IAP 수익 유도 (시간 압박 → 힌트 구매)
3. 세션 종료 트리거 (타이머 없으면 접속만 유지, 목숨이 계속 유지됨)

### 이동 횟수 제한으로 전환하는 이유

| 관점 | 타이머 | 이동 횟수 제한 |
|------|--------|--------------|
| 압박 유형 | 시간 압박 (스트레스) | 전략적 압박 (몰입) |
| 수익 전환 | 광고 +30초 위주 | 힌트·IAP·광고 3중 구조 |
| 감정 유발 | 불안 | "아깝다" → 자발적 구매 |
| 캐주얼 친화 | 낮음 | 높음 |
| 세션 종료 | 타이머 만료 | 이동 횟수 소진 |

### 이동 횟수 동작 규칙
- **성공한 연결(합 = 타겟)만 횟수 차감** (1회)
- **실패한 시도(숫자 배치만)는 차감 없음** — 수학 퍼즐 성격 유지
- 횟수 소진 시 게임오버 팝업 → 3가지 선택지

### 구현 범위 (프로덕션 승인 후)
- 수정 대상: `startTimer()`, `updateTimer()`, `_onTimerEnd()`, `_claimTimeExtend()`
- 추가 대상: `G.movesLeft` 상태, 연결 성공 시 `movesLeft--`
- DESTINATIONS 11개 행성: `timeLimit` → `moveLimit` 속성 교체
- 예상 수정량: ~100~150줄

## 4. 게임오버 팝업 설계

이동 횟수 0 소진 시 자동 표시. 3가지 선택지 제공.

```
┌────────────────────────────┐
│     이동 횟수 부족!         │
│  계속하려면 이동 추가하세요  │
├────────┬────────┬──────────┤
│  ⭐×3  │ ₩1,200 │  📺 광고  │
│힌트→+5회│ +15회  │  +3회 무료│
└────────┴────────┴──────────┘
│     처음부터 다시 (무료)     │
```

- 힌트 소비 → 이동 전환: 기존 힌트 재고 소진 유도
- IAP 직결: 게임오버 시점에 결제 의향 최고조
- 광고: 결제 거부 사용자 수익화
- "처음부터" 옵션 항상 제공 → 강압감 제거
