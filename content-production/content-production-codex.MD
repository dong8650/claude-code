# 콘텐츠 생성 자동화 파이프라인 — Codex 정리 문서

> 이 문서는 `content-production/` 전체를 Codex 관점에서 다시 정리한 운영/전략 문서다.
> 기존 자동화 히스토리, 현재 진단, 앞으로의 콘텐츠 방향성을 한곳에 남긴다.

---

## 1. 현재 결론

`content-production` 파이프라인은 기술적으로 실패한 프로젝트가 아니다.

오히려 다음 요소는 이미 꽤 높은 수준으로 구현되어 있다.

- GPT/Claude 기반 대본 생성
- Quality Gate 기반 대본 필터링
- DALL-E 이미지 생성
- Pexels 스톡 영상 기반 다큐형 생성
- Edge TTS / ElevenLabs 기반 음성 생성
- ASS 자막 생성
- FFmpeg 기반 쇼츠 영상 합성
- PIL 기반 인포그래픽 생성
- n8n 기반 일일 자동 생성 및 YouTube 업로드
- Slack 알림
- 메인/백업 서버 이중화

다만 멈춘 이유는 기술 문제가 아니라 콘텐츠 방향 문제다.

현재 시스템은 어느 순간부터 `매일의 설계`라는 채널 정체성보다
`쇼츠 알고리즘 통과`, `충격 hook`, `저장/공유 유도`, `정체성 공격`에 더 강하게 최적화되었다.

그래서 자동화가 잘될수록 더 많은 영상이 만들어졌지만,
만드는 사람 입장에서는 "내가 정말 이걸 말하고 싶었나?"라는 감각이 약해졌다.

핵심 진단:

> 엔진은 좋다. 핸들을 다시 잡아야 한다.

## 2. 외부 피드백 이후 상태

Threads 앱에서 유튜브 구독자 증가 방법을 물었고,
다음과 같은 강한 피드백을 받았다.

> "죄송하지만 문제가 너무 많아서 어디서부터 얘기해야 할지를 모르겠습니다"

이 피드백 이후 채널 운영이 멈췄다.

Codex 판단:

- 채널이 망한 것은 아니다.
- 문제가 너무 많아서 손쓸 수 없는 상태도 아니다.
- 다만 화면, 주제, 문장 톤, 자동화 기준이 서로 다른 방향을 보고 있었다.
- 조회수 2천대 영상이 나온 것은 Shorts 피드 테스트를 일부 통과했다는 의미다.
- 하지만 구독할 이유와 채널의 반복 가치가 약했다.

즉 문제는 "품질이 전부 낮다"가 아니라 다음에 가깝다.

> 개별 영상은 볼 수 있지만, 채널 전체가 무엇을 약속하는지 선명하지 않다.

## 3. 기존 채널 정체성

채널명:

| 항목 | 내용 |
|---|---|
| 채널명 | 매일의 설계 |
| 핸들 | `@life-architecture` |
| 기존 설명 | 어제보다 나은 오늘을 살고 싶은 분들을 위한 채널 |
| 기존 미션 | 대신 말해주고, 관점을 바꿔주는 채널 |
| 대상 | 30~40대 직장인 |
| 주제 | 감정, 철학, 현실, 돈, 인간관계, 건강 |

기존 미션은 나쁘지 않다.
하지만 너무 넓고, 자동화 규칙에서는 점점 "대신 말해주기"가 "세게 찌르기"로 변했다.

## 4. 새 채널 방향

앞으로 `매일의 설계`는 다음 방향으로 고정한다.

> 30~40대 직장인이 일, 돈, 몸을 무너지기 전에 다시 설계하는 채널

또는 더 짧게:

> 흔들리는 30~40대를 위한 현실 설계 노트

이 채널은 다음이 아니다.

- 명언 채널
- 독설 채널
- 자기계발 구호 채널
- 공포 마케팅 채널
- AI 이미지 쇼츠 공장
- 무작위 정보 큐레이션 채널

이 채널은 다음이다.

- 30~40대 현실을 짧고 선명하게 정리하는 채널
- 일, 돈, 몸을 다시 설계하도록 돕는 채널
- 시청자의 머릿속을 차분히 정리해주는 채널
- "맞아, 내 얘기다"에서 끝나지 않고 "그래서 뭘 먼저 봐야 하지?"까지 남기는 채널

## 5. 새 콘텐츠 축

앞으로 콘텐츠는 3개 축만 사용한다.

### 5.1 일의 설계

다루는 범위:

- 회사
- 말하기
- 상사/동료
- 인정욕구
- 퇴사 고민
- 번아웃
- 업무 경계선
- 직장 내 자기보호

예시 주제:

- 착한 사람이 회사에서 먼저 망가지는 이유
- 직장에서 말이 짧아져야 하는 순간
- 퇴사보다 먼저 봐야 할 신호
- 인정받으려다 무너지는 사람의 패턴

### 5.2 돈의 설계

다루는 범위:

- 월급
- 고정비
- 저축
- 집
- 빚
- 노후
- 퇴직금
- 생활비 구조

예시 주제:

- 월급 300이 위험한 진짜 이유
- 10년 모아도 집 못 사는 구조
- 고정비가 늘면 노력이 먼저 무너진다
- 노후 준비보다 먼저 봐야 할 숫자

### 5.3 몸의 설계

다루는 범위:

- 수면
- 피로
- 자세
- 목/허리
- 운동
- 회복
- 40대 신체 변화
- 스마트폰 습관

예시 주제:

- 8시간 자도 피곤한 이유
- 목이 망가지는 건 자세 문제가 아니다
- 달리기보다 먼저 봐야 할 몸의 신호
- 40대 몸은 의지가 아니라 회복력의 문제다

## 6. 버릴 것과 살릴 것

### 6.1 버릴 것

- `넌 겁쟁이야` 같은 정체성 공격형 hook
- `이거 저장 안 하면 또 당한다` 같은 협박형 CTA
- `지금 주변에 퍼뜨려야 함` 같은 과한 공유 유도
- 가상 출처 기반 인포그래픽
- 근거 없는 수치
- 너무 AI스러운 우주, 뇌, 철학 이미지
- 주제가 넓어 보이는 무작위 업로드
- 매일 2편 완전 자동 업로드
- 조회수만 보고 좋은 영상이라고 판단하는 방식

### 6.2 살릴 것

- 30~40대 현실감
- 일/돈/몸이라는 생활 밀착 주제
- 짧고 선명한 문장
- 다크 오렌지 브랜드 톤
- 자동화 파이프라인
- Quality Gate 구조
- n8n 운영 구조
- YouTube Analytics 분석 구조
- 인포그래픽 포맷

## 7. 새 영상 공식

앞으로 모든 쇼츠는 아래 구조를 기본으로 한다.

```text
1초: 현실 문제를 콕 집는다
5초: 왜 그런지 설명한다
10초: 시청자가 자기 얘기로 느낀다
마지막: 하나의 설계 원칙을 남긴다
```

나쁜 방향:

```text
넌 이미 망했다.
이거 저장 안 하면 또 당한다.
```

좋은 방향:

```text
월급 300이 위험한 이유

돈을 못 벌어서가 아니다.
매달 구조가 새고 있다.
고정비가 늘면
노력보다 먼저 무너진다.

먼저 새는 곳부터 막아라.
```

핵심은 세게 때리는 것이 아니라 정리해주는 것이다.

## 8. 새 대본 톤

앞으로 대본은 다음 기준을 따른다.

### 8.1 문장 톤

- 단문
- 구체적
- 차분함
- 현실적
- 판단은 선명하지만 모욕적이지 않음
- 시청자에게 죄책감보다 구조 이해를 줌

### 8.2 금지 톤

- 과한 독설
- 정체성 모욕
- 협박형 CTA
- 막연한 명언
- 허세 섞인 철학
- 불안만 남기는 엔딩

### 8.3 권장 엔딩

기존:

```text
이거 저장 안 하면 또 당한다
```

새 방향:

```text
먼저 구조부터 봐라
오늘 하나만 줄여라
회복도 계획이다
말보다 경계가 먼저다
```

## 10. 기존 n8n 자동화 히스토리

기존 목표:

> 매일 00:00에 인포그래픽 1편 + 내레이션/다큐 1편을 자동 생성하고 YouTube에 업로드한다.

기존 스케줄:

| 요일 | 인포그래픽 | 에피소드 |
|---|---|---|
| 월/수/금/일 | 생성 | 내레이션형 |
| 화/목/토 | 생성 | 다큐형 |

기존 흐름:

```text
00:00 Cron
  ↓
Daily Plan
  ↓
Git Sync
  ↓
Generate Infographic Data
  ↓
Infographic Generate
  ↓
YouTube Infographic Upload
  ↓
Episode Generate
  ↓
Wait
  ↓
Get Episode Info
  ↓
YouTube Episode Upload
  ↓
Slack Success/Failure
```

평가:

- 기술 자동화는 잘 되어 있다.
- 운영 위험도 고려되어 있다.
- 백그라운드 실행, Wait, Slack 알림도 잡혀 있다.
- 하지만 현재 방향에서는 완전 자동 업로드를 바로 켜면 안 된다.

현재 권장:

> n8n은 생성까지만 사용하고, 공개 업로드는 사람이 승인한다.

## 11. 채널별 히스토리 요약

이 문서는 오늘의 전략만 담는 문서가 아니라,
`content-production/` 전체가 어떤 시행착오를 거쳐 현재 구조가 되었는지 보존해야 한다.

원본 히스토리 문서:

| 영역 | 원본 문서 |
|---|---|
| 마인드셋 | `content-mindset/CLAUDE.md` |
| 건강 | `content-health/CLAUDE.md` |
| 공통 아키텍처 | `content-pipeline-core/CLAUDE.md` |

아래 내용은 세 문서에서 Codex 관점으로 필요한 히스토리를 압축한 것이다.

### 11.1 content-mindset 히스토리

`content-mindset`는 `매일의 설계` 메인 채널의 첫 번째 완성형 파이프라인이다.

핵심 흐름:

```text
3가지 영상 스타일 완성
  ↓
n8n YouTube 업로드
  ↓
일일 자동화
  ↓
Git Sync 구조
  ↓
인포그래픽 AI 생성
  ↓
서버 이중화
  ↓
런타임 경로 분리
  ↓
YouTube Analytics 분석
  ↓
좋아요/저장 CTA 전략 강화
```

주요 변천:

| 버전/날짜 | 의미 |
|---|---|
| 2026-05-02 v3.2 | 내레이션형, 인포그래픽형, 다큐형 3가지 영상 스타일 완성 |
| 2026-05-03 v3.3 | n8n YouTube 자동 업로드 실전 검증, 첫 비공개 업로드 성공 |
| 2026-05-03 v3.4 | 매일 00:00 인포그래픽 + 내레이션/다큐 자동 생성 구조 완성 |
| 2026-05-03 v3.5 | 서버→n8n 콜백 루프 제거, 단일 직선 파이프라인 완성 |
| 2026-05-03 v3.6 | Git Sync 구조 완성, 코드 관리=git / 서버=실행 환경 구조 확립 |
| 2026-05-03 v3.8 | Claude API 기반 인포그래픽 데이터 자동 생성 완성 |
| 2026-05-04 v3.9 | 메인 서버 자동화 첫 실행 성공, 백업 서버 세팅 완료 |
| 2026-05-05 v4.0 | `/root/auto_pipeline` 의존 제거, git repo 직접 실행 구조 완성 |
| 2026-05-05 v4.1 | YouTube Analytics 분석, 인포그래픽 헤더 개선, 공통 브랜딩 연결 |
| 2026-05-05 v4.2 | CTA 오버레이와 closing 4패턴으로 좋아요/저장 전략 강화 |

중요한 성취:

- 자동 생성에서 자동 업로드까지 전체 루프가 실제로 동작했다.
- 단일 워크플로우 안에서 인포그래픽과 에피소드를 모두 처리했다.
- `get_episode_info.py`로 n8n이 에피소드 메타데이터를 안정적으로 읽을 수 있게 했다.
- `setsid + </dev/null`로 SSH 노드가 영상 생성 완료까지 붙잡히는 문제를 해결했다.
- `analyze_youtube.py`로 완시율/좋아요율 기반 분석 루프를 시작했다.

현재 재평가:

- 기술 완성도는 높다.
- 하지만 v4.2에서 좋아요/저장 유도와 공격형 hook 최적화가 너무 강해졌다.
- 새 방향에서는 이 히스토리를 폐기하지 않고, 자동화 뼈대만 살려 콘텐츠 철학을 바꿔야 한다.

### 11.2 content-health 히스토리

`content-health`는 처음부터 건강 채널로 시작한 것이 아니라,

공통 아키텍처 핵심:

코드:

중요한 성취:

### 11.4 히스토리에서 얻은 교훈

이 프로젝트가 지나온 길은 단순히 "자동화 만들기"가 아니었다.

반복해서 해결한 문제:

- 생성은 되는데 업로드가 불안정한 문제
- 영상은 만들어지는데 TTS/자막 싱크가 어긋나는 문제
- 서버별 파일 경로가 꼬이는 문제
- n8n SSH 노드가 백그라운드 작업을 제대로 반환하지 않는 문제
- 이미지 생성 비용과 품질의 균형 문제
- 알고리즘 1차 테스트는 통과하지만 2차 확산에서 멈추는 문제
- 채널별 브랜딩이 중복 정의되는 문제
- 자동화 속도가 콘텐츠 방향성보다 앞서가는 문제

최종 교훈:

> 자동화는 이미 충분히 강하다.
> 이제 필요한 것은 더 빠른 생산이 아니라 더 정확한 방향이다.

## 12. 기존 코드 진단

### 12.1 ai_orchestrator.py

현재 `CONTENT_RATIO`:

```python
CONTENT_RATIO = {
    "emotion": 30,
    "ranking": 30,
    "money":   20,
    "quote":   20,
}
```

문제:

- 새 방향인 `일/돈/몸`과 맞지 않는다.
- `emotion`, `quote`가 너무 넓다.
- 채널 정체성보다 감정 자극 중심으로 흐를 수 있다.

개선 방향:

```python
CONTENT_RATIO = {
    "work": 40,
    "money": 30,
    "body": 30,
}
```

### 12.2 generate_script.py

현재 문제:

- `emotion` 타입이 `정체성 공격형`으로 설계되어 있다.
- hook에 `너/넌/니`를 강제한다.
- closing이 좋아요/저장/공유 유도 중심이다.
- 콘텐츠가 설계형보다 공격형으로 나올 확률이 높다.

개선 방향:

- `정체성 공격형` 제거
- `현실 설계형` 도입
- hook은 문제를 콕 집되 모욕하지 않는다.
- closing은 CTA가 아니라 설계 원칙으로 끝난다.

### 12.3 quality_gate.py

현재 평가 기준:

- scroll stop
- emotional attack
- repeat value
- view score

문제:

- 채널다움 평가가 없다.
- 신뢰도 평가가 없다.
- 실제 도움이 되는지 평가가 없다.
- 구독 이유가 생기는지 평가가 없다.

개선 방향:

새 점수 추가:

| 점수 | 의미 |
|---|---|
| `identity_fit` | 매일의 설계다운가 |
| `trust_score` | 과장/가짜 출처 위험이 없는가 |
| `practical_value` | 보고 나서 하나라도 정리되는가 |
| `subscribe_reason` | 다음 편도 볼 이유가 생기는가 |

### 12.4 generate_infographic_data.py

현재 문제:

```text
source 신뢰감 있는 출처 (직장인 설문, 통계청, 연구소 등 — 가상 가능)
```

이 부분이 위험하다.

돈/건강/통계 콘텐츠에서 가상 출처는 장기적으로 신뢰를 깎는다.

개선 방향:

- `가상 가능` 제거
- 출처가 불확실하면 `source_note`로 "추정/예시/일반화" 명시
- 실제 통계 기반 콘텐츠와 의견형 콘텐츠를 분리

### 12.5 make_video.py / make_video_stock.py

현재 특징:

- 상단/하단 검은 바
- 다크 오렌지 제목
- 워터마크
- 슬로건
- CTA 오버레이

문제:

- 모든 콘텐츠가 같은 압박감으로 보인다.
- 건강/몸/회복 주제에도 공격적 CTA가 붙는다.
- "설계"보다 "쇼츠 마케팅" 느낌이 날 수 있다.

개선 방향:

- CTA 문구를 채널 원칙형으로 변경
- 몸의 설계는 조금 더 차분한 톤 허용
- 영상 타입별 visual tone 분리

## 15. 새 프롬프트 방향

대본 생성 시스템 프롬프트는 다음 방향으로 바꾼다.

```text
당신은 30~40대 직장인을 위한 현실 설계형 쇼츠 작가입니다.
채널명은 매일의 설계입니다.

역할:
- 시청자의 현실을 과장 없이 짚는다.
- 문제를 개인 탓으로 몰지 않고 구조로 설명한다.
- 마지막에는 하나의 설계 원칙을 남긴다.

금지:
- 정체성 모욕
- 협박형 CTA
- 가짜 통계
- 막연한 명언
- 과한 불안 조장

필수:
- 일/돈/몸 중 하나의 축이 분명해야 한다.
- 첫 문장은 현실 문제를 콕 집어야 한다.
- 본문은 왜 그런지 짧게 설명해야 한다.
- 마지막은 실천 가능한 관점으로 끝나야 한다.
```
